Kanban Agile Methodology:-

Kanban is an Agile Software development methodology that uses a visual workflow management system to optimize that flow of work. It is based on a pull system whwre tasks are pulled only when there is capacity, aiming to limit. Work-In-Progresss (WIP) and deliver business value faster.


Key Concepts:
  • Work-In-Progress(WIP):
    • Limit task in progress at each stage to prevent overload.
  • Kanban Board:
    • Visual represention of workflow; Columns = stages, cards = tasks
  • Pull System:-
    • Teams pull work when already rather than pushing tasks.
  • Lead Time:-
    • Total time from task request to completion.
  • Cycle Time:-
    • Time spent actively working on a risk.

Principles:
  1. Start with the exiting process:-
    2. Incremental and evolutionary changes.
  2. Continuous Improvement:-
    3. Small incremental changes over time.
  3. Respectt current roles and responsibilites:-
    4. do not force role changes.
  4. Leadership at all levels:-
    5. everyone encouraged to take initiative.

Features:
  • Visualize workflow stages using bords and cards.
  • Limit WIP to maintain smooth task flow.
  • Flexible and adaptive to maintain smooth task flow.
  • Flexible and adaptive to changing priorities.
  • Continuous delivery and feedback loops.

Advantages:
  • Improves focus and productivity.
  • Fast delivery of tasks and business value.
  • Transparent workflow , easy to track bottlenecks.
  • Minimizes waste and reduce process overhead.

Disadvantages:
  • Dependencies between tasks can complicate workflow.
  • Less structured than Scrum, may not suit deadlines.

Stream Programming Agile Methodology:

Stream programming in Agile oftan called Value Stream Agile is an approach where work is organized into continuous streams of value that flow through the development process.


Key Concepts:-
  • Value Stream:
    • A sequence of activites that deliver specific business value.
  • Continuous Flow:-
    • Tasks move through the system as soon as capacity is available.
  • Pull System:-
    • Teams pull task from the backlog based on priority and capacity.
  • Lead Time & Cycle Time:-
    • Lead Time = Time from request to delivery; cycle time = time actively spent working on a task.
  • Metrics:-
    • Flow efficience WIP throughput bottlenecks and process predictability are tracked.

Principles:
  1. Organize work arround value streams instead of teams slios.
  2. Deliver small, continuous increments of value rather than waiting for big releases.
  3. Optimize flow across teams and departments.
  4. Continuous improvement using metrics and feedback.
  5. Encourage team automomy and collaboration within and across streams.

Features:-
  • Work organized by value streams, not by individual tasks only.
  • Pull-based execution, limiting WIP per stream.
  • Continuous delivery and rapid feedback.
  • Read-time monitoring using visual boards and metrics.
  • Support multi-team collaboration on complex projects.
  • Works with DevOps and automated pipelines.

Advantages:
  • Faster delivery of business value.
  • High adaptability to changing pirorities.
  • Better visibility of workflow across multiple teams.
  • Bottlenecks are easier to identify and fix.

Disadvantages:-
  • Requries strong coordination between streams and teaams.
  • Dependencies can complicate flow if not managed properly.
  • Less structured than scrum;not ideal for very small teams.
