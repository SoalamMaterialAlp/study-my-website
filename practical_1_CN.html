Practical No.-1

Aim:-
To understand and practically observe data travels through different OSI layer using Cisco Packet Tracer.
Requirement:-

  • Computer System or laptop with network.
  • Wi-Fi / ethernet
  • Cisco Packet Tracer Software.
Theory:-

The OSI (Open System Interconnection) model divides the communication process into seven layers to standardize network communication. Each layer performs a specific-functions and interacts only with its adjacent layers.

Layer No. Layer Name Function
7 Application Layer Provides network services to end-users (e.g HTTP,FTP)
6 Presentation Layer Data translation compresssion encryption
5 Session Layer Controls dialog between systems
4 Transport Layer Ensures reliable delivery UDP, TCP
3 Network Layer Logical addressing and routing IP
2 Data Link Layer Physical addressing MAC framing
1 Physical Layer Transimission of bits through the medium
Procedure:-
Using Cisco Packet Tracer
1.Open Cisco Packet Tracer
  • Launch the cisco packet tracer software on your computer.

2.Create a simple network topology.
  • Drag and drop two PCs.

  • Drag and drop 1 hub from network devices.

  • Connect both PCs to the hub using copper straight-Through cables.

3.Assign IP address
  • Click on PCO desktop IP configuration

IP Address: 192.168.1.
Subnet Mask: 255.255.255.0

4.Switch to simulate mode
  • At the bottom-right of packet tracer select. Simulate instead of real-time.

  • Again-run the Ping command.

5.Analyze packet flow: -
  • In hub topology packets are broadcast to all parts.

Even if only PC1 is the destination the Hub forward the packet to all connected devices.

Result:-

The experiment successfully demonslated how data travel through a Hub in the OSI model unlike a switch the hub sends packet to all connected devices showing the broadcast nature of Hub based.
