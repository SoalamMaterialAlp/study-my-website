Question 1:- What are the various operation on Stack?
Answer:-Stack:-
  • Stack is a type of linear data structure .
  • It follows the LIFO (Last In, First Out) principle.
  • Implementation of this stack can be done by contiguous memory location which is an array.
  • In linked list implementation, stack element are stored in non-contiguous memory.
  • All operation are performed at one end called top.

    Operation on STACK:-

1.Push()
2.Pop()
3.Peek()
4.IsFull()
5.IsEmpty()

    1.Push()

  • Push is a basic stack operation.
  • It is used to insert a new element into the stack.
  • The element is always added at the top of the stack.
  • Before pushing, the stack is checked for overflow.
  • If the stack is full, push operation cannot be performed.
  • When pushing, the top is incremented.
  • The new element is placed at the top position.

    2.Pop()
  • Pop is a basic stack operation.
  • It is used to deletion or remove an element from the stack.
  • The element is always removed from the top of the stack.
  • Before popping, the stack is checked for underflow.
  • If the stack is empty, pop operation cannot be performed.
  • During popping, the value at top is removed.
  • After removal, top is decremented by 1.

    3.Peek()
  • Peek operation () is a basic stack operation.
  • It is used to view the top element of the stack.
  • The value of top remains unchanged after peek.
  • Before peek, the stack is checked for underflow.
  • If the stack is empty peek() cannot be performed.
  • Peeks help to know which element will be popped next.

    4.IsFull():-

  • Isfull operation() is a stack operation used to check whether the stack is full.
  • It is mainly used in array-based implementation.
  • If top == Max-1 , the stack is consider full.
  • It helps in safe Insertion of elements into the stack.
  • If the stack is full then return true.
  • If the stack is space or not full them return false.

    5.IsEmpty():-

  • Isempty () is a basic stack
  • It is used to check whether the stack is empty or not.
  • Isempty helps to avoid stack overflow.
  • If top==-1, the stack is consider empty.
  • When the stack is empty,pop operation cannot be performed.

Question 2:-What is the basic difference between Stack and Queue?
Answer:-The basic difference between Stack and Queue:-

Basis Stack Queue
Definition Stack is a linear data structure that follows LIFO principle. Queue is a linear data structure that follows FIFO principle.
Principle LIFO (Last In, First Out) FIFO (First In, First Out)
Insertion Operation Element is inserted using Push operation at the top. Element is inserted using Enqueue operation at the rear.
Deletion Operation Element is removed using Pop operation from the top. Element is removed using Dequeue operation from the front.
Access Point Insertion and deletion both happen at one end (Top). Insertion and deletion happen at two different ends (Front and Rear).
Pointers Used Uses only one pointer (Top). Uses two pointers (Front and Rear).
Order of Removal Last inserted element is removed first. First inserted element is removed first.
Example(Real Life) Stack of plates. Line of people waiting for tickets.
Applications Function calls, recursion, expression evaluation. CPU scheduling, printer queue, task scheduling.
Complexity Push and Pop operations are O(1). Enqueue and Dequeue operations are O(1).


Question 3:-What are advantage and disadvantage of Stack?

Answer:-Stack:-
  • Stack is a type of linear data structure.
  • It follows LIFO (Last In,First Out) principle.
  • Insertion operation in stack is called push()
  • Deletion operation in stack is called pop()
  • Peek() operation is used to see the top element.
  • Insertion and deletion both happen at one end(top).
  • Overflow occurs when stack is full.
  • Underflow occurs when stack is empty.

Operation of STACK
  • Push()
  • Pop()
  • Peek()
  • isFull()
  • isEmpty()

Advantagesof Stack
  • Simple Structure – Stack is easy to understand and implement.
  • Easy Implementation – Can be implemented using array or linked list.
  • LIFO Principle – Works on Last In First Out method.
  • Fast Operations – Push and Pop operations take O(1) time.
  • Efficient Memory Use (Linked List) – In linked list implementation, memory is used dynamically.
  • Function Call Management – Used in managing function calls (Call Stack).
  • Recursion Support – Recursion uses stack internally.
  • Expression Evaluation – Used in postfix and prefix expression evaluation.
  • Expression Conversion – Infix to postfix and prefix conversion.
  • Parenthesis Checking – Used to check balanced parentheses in expressions.
  • Backtracking – Used in problems like maze solving and DFS.
  • Undo/Redo Operations – Used in text editors.
  • Syntax Parsing – Used in compilers for syntax checking.
  • Reversing Data – Helps in reversing strings or numbers.
  • Browser History – Used to manage backward and forward navigation.

Disadvantages of Stack
  • Fixed Size (Array Implementation) – Size cannot grow dynamically in array method.
  • Stack Overflow – Occurs when pushing element in full stack.
  • Stack Underflow – Occurs when popping from empty stack.
  • No Random Access – Cannot access middle elements directly.
  • Limited Flexibility – Only LIFO order is allowed.
  • Memory Wastage (Array) – Unused space if stack not full.
  • Not Suitable for FIFO Tasks – Cannot be used where First In First Out is required.
  • Possible Memory Issues – Deep recursion may cause stack overflow.
  • Difficult for Large Data Handling – Not ideal for handling large data-sets.
  • Sequential Access Only – Elements can only be accessed from top to bottom.
